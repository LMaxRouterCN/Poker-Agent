# 待解决问题清单

## 1. 【严重】去重机制误杀当前对话指令

### 现象
当前对话中，两条不同的指令（开头相同但内容不同）被当作重复指令跳过。
日志证据：

        
      


08:30:38🎉 捕获指令: find 低配版Agent.js 【CodeSTART】 _dispatch(batch); } }, 6… (共 135 字符)
08:30:39⏭️ 跳过重复指令: find 低配版Agent.js 【CodeSTART】 …






        

          
第一条被捕获，第二条被跳过，但它们是不同的 find 指令。

### 根因分析
流式输出期间，LLM 逐步生成文本。`getCleanText()` 每次轮询拿到的是"当前时刻的快照"。
当 LLM 正在写第二个 【cmd】块时，其内容可能恰好与某个中间状态的第一个块 `fullMatch` 完全一致（比如都在生成开头那段），导致被 `_sentCmds` 误判为重复。

### 期望行为
- 当前对话的指令：只防流式输出中同一条指令被反复执行（用 _sentCmds），不同指令绝不误杀
- 上个对话的指令：新对话时 _sentCmds 已 clear，完全不管，不比对

### 可能的解决方向
不能按完整文本匹配，改为按"指令特征"去重，比如取 cmdStr 的前N个字符+hash，或者只在"同一轮稳定检测"内去重而非跨轮询累积。也可以考虑去掉 _sentCmds 的全文匹配，改用更宽松的策略。
